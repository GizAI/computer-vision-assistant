# Autobot 요구사항

- 이 도구를 확장해서 FastAPI, Vue 3, shadcn-vue로 채팅 인터페이스를 제공하는 Desktop기반의 자율 AI 에이전트인 Autobot을 개발하자. 이 도구는 개인정보보호를 위해서 LLM API를 제외하고 모든 것을 PC에 저장하고 돌린다.
- AI 채팅을 통해서 사용자는 거대한 작업을 요청할수 있다. (예를 들어 아이디어만으로 SaaS같은 비즈니스를 위한 시장조사/기획/개발/런칭/운영, 지능적 쇼핑, 금융 투자, 개인적 업무 관리 등등)
- 초기 프롬프트를 통해서 Autobot은 프로젝트 폴더를 만들고 프로젝트를 시작한다.그러면 Autobot은 작업을 시작해서 자동으로 계속해서 24시간 365일 진행한다.. 인간은 그 작업을 지켜보면서 언제든 프롬프트를 내릴수 있고 Autobot은 해당 프롬프트를 즉시 고려하고 하던 하던 작업을 변경하던지 계속하던지 해야한다.
- 사용자가 요청한 작업을 수행하기 위해서 Autobot은 먼저 목표를 인식하고 계획을 수립한다. 그리고 그 목표를 위해서 끝없이 생각하고 작업을 수행하고 사용자에게 진행상황과 결과를 알려준다. 계획은 트리 구조로 만들어질수 있고 단위 작업 수행을 위해서 중간중간 계획을 세분화 시켜서 업데이트 할수도 있다. 인간이 프로젝트 하는걸 생각해봐라.
- 계획 파일은 프로젝트 폴더에 만들어서 md 파일로 관리하고 작업 진행에 따라서 완료 여부도 체크하고 업데이트 된 계획도 반영한다.
- Autobot이 자신의 계획 수립 과정이나 작업 수행 결과를 스스로 평가하고, "왜 이 방법이 실패했을까?", "더 효율적인 방법은 없었을까?" 와 같은 자기 반성을 통해 다음 작업이나 유사한 프로젝트에서 더 나은 결정을 내리도록 학습하고 다음 작업에 반영한다.
- 로컬 PC에 최적화된 벡터 데이터베이스를 설치하고 OpenAI API의 최신 임베딩 모델로 임베딩을 생성한다. 프로젝트 폴더에 있는 모든 파일과 모든 대화내용에 대한 임베딩을 전략적이고 효과적으로 생성하서 모든 상황에 AI가 능동적으로 참조할수 있게 한다.
- 프로젝트 별 대화내용은 전체가 잘 보관되어야 한다. 이를 위해서 sqlite같은 db가 필요하면 활용한다. 텍스트 파일이 더 적합하면 그것도 괜찮다.
- Autobot은 진행 과정에 얻은 통찰력이나 수집된 정보나 사용자의 입력을 통해서 목표와 계획를 수정할수 있다. 이 경우 계획 파일을 수정해서 업데이트 하고 이후 작업에서 업데이트 된 계획 파일을 참고한다.
- OpenAI API를 연동해서 AI는 이런 작업을 수행해야하는데 이를 위해서 computer_vision_assistant.py를 통해서 컴퓨터 사용능력이 제공된다. CLI 명령어도 마음것 사용할수 있다(단 OS환경에 맞춰서 AI가 제대로된 명령을 생성해야한다다). python 코드를 생성하고 실행하는것도 가능하다. PC상의 모든 권한이 주어지니 가능한 모든 수단을 동원해서 도구로 사용해서 목표를 달성하자. Playwright같은 브라우저 자동화 도구도 사용하자. 브라우저를 사용할때는 사용자가 로그인해둔 사이트를 활용할수 있게 사용자 프로필을 그대로 활용해야한다. UI사용, CLI사용, 웹검색, 웹페이지 읽기, API사용 등등 모든것을 도구로 사용할수 있게 해야한다. AI는 항상 최적의 도구를 적절하게 선택해야한다.
- 간단한 유틸리티성 Python 스크립트나 CLI 명령어 조합 등, 반복적으로 필요한 작업을 자동화하는 도구를 Autobot 스스로 생성했
- AI과 코딩이나 아주 지능적인 에이전트 작업을 위해서 VS Code에서 설치된 Augment Code 확장을 사용할수 있다. 이것은 지능적인 컨텍스트 엔진을 가진 에이전틱 AI 개발도구이다. VS Code는 Electron기반이고 Augment Code는 익스텐션으로 WebView안에 인터페이스가 들어가 있다. 이를 조작할수 있어야 한다.
- Augment Code는 아주 뛰어나지만 작업을 몇시간씩 계속하지는 않고 중간 중간 멈춰서 사용자의 입력을 기다리는 문제가 있으니 Autobot이 Augment Code에게 명령을 내려주고 작업 결과물을 평가하고 지시하고 하는 상사(혹은 인간 사용자)의 역할을 해주어야 한다.
- Augment Code등의 각종 도구의 상태를 더 정확히 파악하고(예: 작업 완료 여부, 현재 진행 상태 등)하고 관리할수 있도록 기록해두고 컨텍스트로 참조할수 있게 해야한다.
- Autobot 자체적으로도 정교한 고성능 컨텍스트 참조 엔진을 개발해서 LLM API호출할떄 어떤 컨텍스를 참조해야할지 잘관리할수 있어야 한다. 한번 열어본 어떤 파일이나 웹페이지 내용이나 대화 내용 등등은다 벡터화 시켜서 잠재적으로 참조할수 있는 컨텍스트(일시 등 메타정보도 넣고)가 되어야 한다.
- 이를 통해서 LLM API 호출시에 최적의 컨텍스트만을 제공할수 있어야 한다. 대화내용에는 방대한 진행상황이 다 들어가 있으므로 전체를 LLM에 넘기지 말고 적절하게 필요한것만 컨텍스트로 넘겨야하는데 최근 대화는 무조건 포함하고, BM25와 HNSW를 통해 의미적·문자적 유사성이 높은 과거 기억을 동적으로 검색하여, 시간순으로 정렬된 최소 맥락을 구성한 뒤 모델 입력으로 전달한다.
- https://github.com/anthropics/claude-code, https://github.com/openai/codex 이런 프로젝트 참고한다.

# 지시사항

프로젝트 UI를 개선하자 UI는 기본적으로 AI채팅인터페이스가 되어야 하는데 2단으로 구성해야한다.. 왼쪽에는 사용자가 AI와 채팅할수 있는 인터페이스를 넣고, 오른쪽에는 AI가 영원히 작업하고 있는 기록을 실시간으로 볼수 있어야 한다.. 그러니까 AI가 사용자 응대 용이랑, 작업용 2가지가 돌아가는거다.. 이런 구조를 고려해서 UI도 변경하고 서버의 핵심로직도 변경되어야 하면 변경해줘.. 영어로 답해.

계속해.. 사용자가 할건 채팅입력 뿐이고 나머지는 AI가 다알아서 하는거다. plan, reflect고 하는건 AI가 할일이지 인간은 그냥 채팅만.. 계속해..

채팅 메시지는 엔터를 쳐서 Send가 되어야하고.. 자동으로 최하단으로 스크롤이 되어야 한다..Status와 Goal은 AI Work Logs 위에 좀 작게 배치해라. ChatGPT UI처럼 2개의 AI 채팅이 거의 꽉찬형태로 보이고 모바일에도 최적화 되어야 한다..

메뉴는 상단이 아니라 왼쪽에 들어가게 해달라.. Project를 채팅 세션 처럼 선택할수 있는 형태로하고.. 메뉴는 햄버거 메뉴로서 접을수 있게 ..모바일도 최적화.. Projects화면은 별도로 필요없고 하나로 통합해라.. Settings도 모달로 뜨게 하라.. 즉 기존의 메뉴는 전부 필요없어지고 통합 UI로.. 그리고 height도 ChatGPT처럼 100%로 고정되게 해라.. width도 100%로하라..

Refresh버튼은 모두 제거하라.. ws을 통해서 실시간으로 업데이트 되어야한다.

Chat with AI라벨 대신에 그냥 프로젝트 이름이 나오게 하라..
AI Work logs제목 대신에 State와 Task와 Goal이 적당히 배치되게하고 일시 중지와 재개할수 있는 버튼을 넣어라..
모든 것은 백엔드와 연동해서 잘돌아가야한다.. 영어로 답해.

AI Work Logs 제목은 필요없고 그자리에 Goal이 배치되게 하고 Goal은 사용자가 클릭해서 사용자가 직관적으로 직접 수정할수 있게하라..
State는 Pause 버튼 왼쪽에 나오게하고..
Task도 적당하게 배치하라.. 현재 plan전체를 확인할수 있는 버튼도 있어야 하는데.. 접어놯다가 펼칠수 있으면 좋겠다.. plan을 사용자가 수정할수도 있게하라..
잠깐 Goal표시를 포함해서 모든것은 라벨을 별도로 주지 말도 더 직관적인 형태로하라.. 가장 모던한 UX가 어떤 것인지 고민해봐라..

프로젝트 시작시는 오케스트레이터가 정지상태로 시작해야한다..
아주것도 없는 상태에서 사용자가 채팅을 시작하면 그때 프로젝트를 생성하고 프로젝트 이름을 LLM으로 알아서 정해서 넣어줘야한다.. 프로젝트 이름앞에 이모지 하나 붙여라. 채팅의 Send는 엔터로도 할수있게하고..
프로젝트의 작업 공간도 볼수 있어야 한다. 서버에 있는 프로젝트 작업공간 파일들을 Windows 탐색기 처럼 보여주고 열어보고 수정하고 할수있어야 한다.. 작업공간은 접어놯다가 펼쳐서 볼수있게 해라..

- 프로젝트 클릭하면 그냥 프로젝트 데이터가 로드되어야 한다.. 지금은 /proejct 메시지가 날아가는데.. 그러지말자.
- New Project버튼은 위로 올리고..클릭하면 별도의 모달이 뜰필요없고 그냥 새 프로젝트가 생기면 되는데.. 실제로 프로젝트 생성은 사용자가 첫번째 메시지를 입력하면 서버에서 제목을 정하면 그때 생성하는거다.. 사용자 메시지 입력에 따라서 AI는 프로젝트 제목, Goal, Plan등을 업데이트할 필요가 있으면 업데이트 하는거다.. 모든건 실시간으로 화면에 반영되어야 한다..

지금 New를 클릭하면 서버에서 새프로젝트를 만들려고 시도하는데.. 그러면 안되고 New를 클릭하면 그냥 프로젝트가 빈상태가 되어야한다. 그리고 사용자가 빈 상태에서 첫번째 메시를 입력하면 그때 프로젝트가 생성되면서 이모지로 시작하는 프로젝트 제목, 목표, 플랜 등을 AI가 업데이트하고 그게 사용에게 실시간으로 보여야 한다..
